SWAT project

  • Parameters defined at the hru level (.hru, .gw, .mgt, .chm, .sdr, .sep, .sol parameters), all columns of this table should be filled.
  • Parameters defined at the subbasin level (.sub, .rte, .wq, .pnd .res parameters), the land use, soil, slope columns should be left empty.
  • Basin parameters (.wwq, .bsn parameters), the subbasin, landuse, soil, slope columns should be left emtpy.
  • If you select all subbasins/landuse/soil or slope, just type 'All'.
  • 'relative' change: xnew = xold × (1 + appliedValue)
  • 'absolute' change: xnew = xold + appliedValue
  • 'replace' change: xnew = appliedValue

SWAT+ project

  • The same formulea as for SWAT were use with 'relative', 'absolute' and 'replace' changes
  • Parameter change can be applied to specific objects (by giving the object number, e.g., HRU number, aquifer number) such as 1:3, 5, 7:8 means for object numbers 1,2,3,5,7,8
  • Parameter change can be also conditioned on several variables to determine which hru’s to apply the changes. Conditions include hydrologic soil group (e.g., 'hsg = A, B, C'), land use (e.g., 'landuse = corn, past'), and soil layer (e.g., 'lyr1 = 1; lyr2 = 2'). Each hru is checked to determine if they match the condition and the change is made to the parameter. Multiple conditions can be applied to each parameter, seperated by semicolons (e.g., 'hsg = A, B, C; landuse = corn, past'). The name of differnt soil hydrologic group (hsg) and land use (landuse) can be found in the 'soils.sol' and 'plants.plt' in your TxtInOut folder.
